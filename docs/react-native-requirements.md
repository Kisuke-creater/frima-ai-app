# Frima AI React Nativeアプリ 要件定義書

- 作成日: 2026-04-11
- 版数: v0.1
- 対象: `Frima AI` のスマートフォンアプリ新規開発
- 前提: 既存Webアプリの主要機能を React Native でモバイル最適化して提供する

## 1. 目的

フリマ出品作業を支援する既存Webアプリ `Frima AI` を、iOS / Android のスマートフォンアプリとして提供する。  
本アプリでは、商品画像の登録、AIによる出品文生成、商品管理、価格交渉支援、利益シミュレーション、市場分析をモバイルで完結できることを目的とする。

## 2. 背景

既存Webアプリでは以下の業務を支援している。

- メールアドレス / パスワードログイン、Googleログイン
- 商品画像をもとにしたAI出品文生成
- 商品一覧管理
- 価格交渉の判断支援と返信文作成
- 利益シミュレーション
- 市場分析
- テーマ / 言語 / アカウント設定

スマートフォンアプリ化により、出先での商品撮影から出品判断までを一連の流れで扱えるようにする。

## 3. 開発方針

- フレームワークは React Native を採用する
- 実装基盤は `Expo` を第一候補とする
- 認証・データ保存は既存Web版と同様に Firebase を継続利用する
- AI生成処理はクライアントにAPIキーを持たせず、既存サーバーAPI相当の安全なバックエンド経由で実行する
- 初期リリースは日本語UIを必須とし、英語UIは将来拡張とする

## 4. 対象ユーザー

- フリマアプリで商品を出品する個人ユーザー
- 中古品販売やせどりを行う小規模ユーザー
- 出品文作成、価格設定、利益計算をスマホ中心で行いたいユーザー

## 5. 対応範囲

### 5.1 対象プラットフォーム

- iOS 16 以上
- Android 10 以上

### 5.2 対象デバイス

- スマートフォンを必須対応
- タブレットはレイアウト崩れなく表示できることを推奨対応とする

### 5.3 リリース範囲

初期リリースでは以下を対象とする。

- 認証
- ダッシュボード
- 商品登録
- 商品管理
- 価格交渉AI
- 利益シミュレーション
- 市場分析
- 設定

## 6. 対象外

初期リリースでは以下は対象外とする。

- フリマサービスへの自動出品連携
- 決済機能
- チャット通知のプッシュ通知
- 管理者向け運用画面
- オフライン完全対応
- 英語UIの本実装

## 7. 利用シナリオ

### シナリオ1: 商品撮影から出品文作成まで

1. ユーザーがアプリでログインする
2. カメラまたは写真ライブラリから商品画像を選ぶ
3. 商品名、付属品、状態、販売先を入力する
4. AIがタイトル、説明文、カテゴリ、価格候補を生成する
5. ユーザーが価格を選択し、商品を保存する

### シナリオ2: 出品後の価格交渉対応

1. ユーザーが出品中の商品を選択する
2. 購入希望者のメッセージを入力する
3. AIが推奨対応、推奨返信価格、利益見込み、返信文案を返す
4. ユーザーが内容をコピーして外部フリマアプリで返信する

### シナリオ3: 利益確認と販売判断

1. ユーザーが商品を選択する
2. 販売価格、サイズ、重量、資材情報を入力する
3. 利益シミュレーションを実行する
4. 販売先ごとの利益比較を確認する

## 8. 機能要件

## 8.1 認証

### FR-01 ログイン

- メールアドレス / パスワードでログインできること
- Googleアカウントでログインできること
- セッション維持に対応すること
- ログアウトできること

### FR-02 新規登録

- メールアドレス / パスワードで新規登録できること
- パスワードは8文字以上を推奨すること
- 登録失敗時に理由を表示すること

## 8.2 ダッシュボード

### FR-03 概要表示

- 総アイテム数を表示できること
- 出品中件数を表示できること
- 販売済み件数を表示できること
- 売上サマリーを表示できること
- 最新の商品一覧を表示できること

## 8.3 商品登録

### FR-04 画像登録

- カメラ撮影で画像を取得できること
- 写真ライブラリから画像を選択できること
- 最大6枚まで登録できること
- JPEG / PNG / WEBP / GIF を受け付けること
- 非対応形式はエラー表示すること
- 画像はアップロード前に圧縮・リサイズできること

### FR-05 AI生成

- 入力項目として以下を受け付けること
  - 商品名
  - 付属品
  - 商品状態
  - 出品先
  - 商品画像
- AI生成結果として以下を返せること
  - タイトル
  - 説明文
  - カテゴリ
  - 価格候補3案
  - 状態補足
- 生成中はローディング状態を表示すること
- 失敗時は再試行可能とすること

### FR-06 商品保存

- AI生成結果をユーザーが編集できること
- 選択価格で商品を保存できること
- 画像は Firebase Storage 等のストレージに保存できること
- 保存データは Firestore 等のデータベースに保持できること

## 8.4 商品管理

### FR-07 商品一覧

- 商品一覧をカードまたはリスト形式で表示できること
- 出品中 / 販売済みで絞り込みできること
- 商品画像、タイトル、価格、状態、出品先を確認できること

### FR-08 商品詳細

- 商品詳細画面で登録内容を確認できること
- 複数画像をスワイプ表示できること
- 販売済み登録ができること
- 販売価格と販売日時を記録できること

## 8.5 価格交渉AI

### FR-09 交渉支援

- 出品中の商品を選択できること
- 想定販売先を選択できること
- 以下の入力を受け付けること
  - 購入希望価格
  - 仕入原価
  - 希望利益
  - 送料
  - 梱包費
  - 購入希望者メッセージ
- AIが以下を返せること
  - 推奨判断
  - 推奨返信価格
  - 想定利益
  - 返信文案
  - 判断理由
- 定型クイックアクションを提供すること
  - 10%オフ相談
  - 20%オフ相談
  - 値下げできますか
  - 早売り3案

## 8.6 利益シミュレーション

### FR-10 利益計算

- 出品中の商品を選択できること
- 以下の入力を受け付けること
  - 販売価格
  - 販売先
  - 縦 / 横 / 高さ
  - 重量
  - 梱包資材
- 全プラットフォーム比較または単一プラットフォーム比較ができること
- 計算結果として以下を表示できること
  - 手数料
  - 送料
  - 梱包費
  - 予想利益
  - 利益順比較
- 入力値は商品ごとに保存できること

## 8.7 市場分析

### FR-11 市場分析

- 商品タイトル、説明、状態、画像枚数をもとに市場分析を実行できること
- AIにより以下の結果を表示できること
  - 需要レベル
  - 推奨販売先
  - 推奨理由
  - 想定価格帯
  - 想定販売日数
  - プラットフォーム比較
- 分析結果は履歴として保存できること

## 8.8 設定

### FR-12 設定管理

- テーマ設定を変更できること
  - ライト
  - ダーク
  - システム追従
- 言語設定を表示できること
  - 初期リリースでは日本語を有効
  - 英語は将来対応として表示のみ可
- アカウント情報を表示できること
- CSVエクスポート / インポートを実行できること

3. 新規登録
4. ダッシュボード
5. 商品登録
6. 商品一覧
7. 商品詳細

## 10. UI / UX要件

- 片手操作を前提に主要操作は画面下部でも押しやすい配置とする
- 商品画像の登録導線は最短3タップ以内とする
- 主要処理中は必ず進行状態を表示する
- 入力エラーはフィールド単位でわかるようにする
- 商品画像、価格候補、利益結果はスマホ画面で一目で比較できること
- iOS / Android の標準操作感に馴染むこと
- ダークモードに対応すること

## 11. 非機能要件

### 11.1 性能

- ログイン後の初期表示は3秒以内を目標とする
- 主要一覧画面は5秒以内に初回表示されること
- AI処理は通信状況に応じてローディング表示を継続すること

### 11.2 セキュリティ

- OpenAI APIキーをアプリ内に保存しないこと
- 認証は Firebase Authentication を利用すること
- 通信は HTTPS を必須とすること
- ユーザーごとのデータ分離を保証すること
- Firestore / Storage ルールでアクセス制御すること

### 11.3 可用性

- 通信失敗時にリトライ導線を設けること
- 一時的なAPI失敗時もアプリが強制終了しないこと

### 11.4 保守性

- Web版と用語・データ構造を極力そろえること
- 画面、API、データアクセス層を分離すること
- ログ収集基盤を導入できる設計にすること

### 11.5 アクセシビリティ

- 主要操作にアクセシビリティラベルを付与すること
- 色だけに依存した状態表現を避けること
- Dynamic Type相当の文字拡大でも利用可能とすること

### 12.1 ユーザー

- UID
- メールアドレス
- 表示名
- 設定情報
  - テーマ
  - 言語

### 12.2 商品

- 商品ID
- ユーザーID
- タイトル
- 説明
- 画像URL配列
- カテゴリ
- 商品状態
- 出品価格
- 出品先
- ステータス
- 販売価格
- 登録日時
- 販売日時
- 配送情報
  - サイズ
  - 重量
  - 梱包資材ID
  - 梱包資材費

### 12.3 分析履歴

- 分析ID
- ユーザーID
- 入力値
- 推奨タイトル / 説明
- 需要レベル
- 推奨販売先
- 推奨理由
- 想定価格帯
- 想定販売日数
- 作成日時

## 13. 外部連携要件

- Firebase Authentication
- Firestore
- Firebase Storage
- AI生成用バックエンドAPI
  - 商品登録AI生成API
  - 価格交渉AI API
  - 市場分析API

補足:
既存Web版の `Next.js API Route` をそのまま利用する場合は、モバイルアプリから安全に呼べる形で HTTPS 公開する必要がある。  
将来的には Firebase Functions などへの移行も選択肢とする。

## 14. 権限要件

- カメラ権限
- 写真ライブラリ権限
- 通信権限

通知機能を追加する場合のみ、将来の拡張として通知権限を扱う。

## 15. 想定技術構成

- フロントエンド: React Native
- 開発基盤: Expo
- ナビゲーション: Expo Router または React Navigation
- 認証 / DB / ストレージ: Firebase
- 状態管理: React Context または Zustand
- フォーム管理: React Hook Form 相当
- API通信: fetch または Axios
- 画像取得: Expo Image Picker / Camera

## 16. MVP定義

初期リリースで最低限満たすべき条件を以下とする。

- ユーザー登録とログインができる
- 商品画像を登録してAI出品文を生成できる
- 商品を保存し、一覧で確認できる
- 利益シミュレーションができる
- 価格交渉AIを利用できる
- 設定画面でテーマと言語表示を確認できる

## 17. 受け入れ条件

- iOS / Android 実機でログインから商品保存まで完了できること
- 画像アップロードからAI生成まで一連で実行できること
- 保存した商品が商品一覧に反映されること
- 利益シミュレーション結果が正しく表示されること
- 価格交渉AIで返信文案が返ること
- 異常系でユーザーが次の操作を判断できるエラー文言が出ること
